# chat-app

A minimal chat web app: FastAPI backend, plain HTML/CSS/JS frontend, no frontend framework.

Replies are generated by a real agent — a read-only "codebase concierge" built on the
[Claude Agent SDK](https://github.com/anthropics/claude-agent-sdk-python) that can read files,
search the repo, and answer questions about it. All of that logic is isolated in one function, so
the agent's behavior can keep evolving without touching the route, response model, or frontend.

## Requirements

- Python 3.12+
- [uv](https://docs.astral.sh/uv/)
- The `claude` CLI available on `PATH` (installed as part of `claude-agent-sdk`'s dependencies)
- Auth for the Claude Agent SDK (e.g. `ANTHROPIC_API_KEY` set, or already logged in via the
  `claude` CLI)

## Run

```bash
uv sync                                   # one-time: install dependencies
uv run uvicorn app.main:app --reload      # start the server
```

Then open <http://127.0.0.1:8000>.

### Configuration

- `TARGET_REPO_PATH` — the repository the concierge answers questions about and can read files
  from. Defaults to this project's own directory. Set it to point the agent at a different repo:

  ```bash
  TARGET_REPO_PATH=/path/to/other/repo uv run uvicorn app.main:app --reload
  ```

## API

`POST /api/chat`

```bash
curl -X POST http://127.0.0.1:8000/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"message": "How many lines are in app/agent.py?", "conversation_id": "test-1"}'
```

```json
{ "reply": "`app/agent.py` has **177 lines**." }
```

Send further requests with the same `conversation_id` to continue the same conversation — the
agent remembers earlier turns in that chat. A new `conversation_id` starts a fresh session with no
memory of other conversations. Any failure talking to the agent (bad `TARGET_REPO_PATH`, CLI not
found, etc.) comes back as a normal 200 response with a polite fallback `reply`, never a 500.

## The agent

Everything that decides *what the assistant says* lives in one place:

```
app/agent.py  →  generate_reply(message, conversation_id) -> str
```

It calls the Claude Agent SDK's `query()` with:

- **System prompt**: instructs the model to act as a read-only codebase concierge for
  `TARGET_REPO_PATH`, answering concisely and citing file paths.
- **Tools**: `Read`, `Glob`, `Grep`, plus two custom tools served in-process via an SDK MCP server
  (`create_sdk_mcp_server`, exposed as `mcp__concierge__*`):
  - `count_lines(file_path)` — line count of a file in the repo.
  - `git_log(n)` — the last `n` commit hashes and subject lines (default 5, max 20).
  
  Both `tools` and `allowed_tools` are set to this same list, so nothing outside it (e.g. `Bash`,
  `Write`, `Edit`) is reachable — this is what makes the agent read-only.
- **`max_turns=25`** — a hard cap per request.
- **Session resumption**: a conversation's SDK `session_id` is captured from the first turn and
  reused on later turns via `resume=`, so the agent has memory within a `conversation_id`. This
  mapping lives in an in-memory dict and resets on process restart — there's no database.
- Every custom tool call is logged server-side (`logger.info` in `app/agent.py`).

`conversation_id` is generated per page load by the browser and passed through on every request —
it's the key into that in-memory session map.

## Layout

```
app/
  main.py      FastAPI app: GET / and POST /api/chat
  agent.py     the agent: generate_reply(), the concierge system prompt, and its two custom tools
static/
  index.html   chat UI
  style.css
  app.js       fetch() calls /api/chat, renders both sides
```
